# EKO-ELSA (인천서부지사 탄소중립 실천 앱)

EKO-ELSA는 한국승강기안전공단 인천서부지사 임직원을 위한 탄소중립 실천 및 환경 보호 독려용 모바일 웹 애플리케이션(PWA)입니다. 게임화(Gamification) 요소를 도입하여 일상 속 환경 보호 활동을 재미있게 실천하고 보상을 얻을 수 있도록 설계되었습니다.

## 🚀 프로젝트 개요

- **목적**: 사무실 내 에너지 절약 및 탄소 저감 활동 활성화
- **주요 기능**:
  - **에코 미션**: 계단 이용, 다회용 컵 사용, 페이퍼리스 활동 등 일상 미션 인증
  - **캐릭터 성장**: 획득한 포인트에 따라 '나무', '북극곰', '펭귄' 캐릭터가 4단계로 성장
  - **탄소 저감 통계**: 개인 및 지사 전체의 누적 탄소 저감량을 실시간으로 확인
  - **지사 랭킹**: 지사원 간의 포인트 경쟁 및 선물 기능
  - **PWA 지원**: 모바일 홈 화면 추가 및 오프라인 접근 가능

## 🛠 기술 스택

- **Frontend**: HTML5, Vanilla JavaScript, CSS3
- **Styling**: [Tailwind CSS](https://tailwindcss.com/) (CDN 버전 사용)
- **Icons**: [Lucide Icons](https://lucide.dev/)
- **PWA**: Service Worker (`sw.js`), Web App Manifest
- **Storage**: 브라우저 `localStorage` (포인트, 미션 상태, 리더보드 데이터 저장)
- **Deployment**: [Vercel](https://vercel.com/) (`vercel.json` 참조)

## 📂 주요 파일 구조

- `index.html`: 애플리케이션의 UI, 스타일, 비즈니스 로직이 포함된 메인 단일 파일
- `sw.js`: 오프라인 지원을 위한 서비스 워커 (기본 정적 파일 캐싱)
- `manifest.txt` / `icons/site.webmanifest`: PWA 설치 및 환경 설정을 위한 매니페스트
- `vercel.json`: Vercel 배포를 위한 캐시 제어 및 헤더 설정
- `icons/`: 앱 아이콘 및 파비콘 리소스

## 🏃 실행 및 테스트

본 프로젝트는 별도의 빌드 과정이 필요 없는 정적 웹 사이트입니다.

### 로컬 실행
1. 프로젝트 루트 디렉토리에서 라이브 서버를 실행합니다.
   - VS Code 사용 시: `Live Server` 확장 프로그램 권장
   - Python 사용 시: `python3 -m http.server 8000`
2. 브라우저에서 `http://localhost:8000`으로 접속합니다.

### 미션 테스트 (QR 코드)
- '폐배터리 수거' 미션은 URL 파라미터 `?mission=m8`을 통해 인증 로직이 트리거됩니다.

## 📝 개발 규칙 및 관례

1. **단일 파일 구조**: 현재 모든 로직이 `index.html`에 포함되어 있습니다. 규모가 커질 경우 JS 로직을 모듈별로 분리하는 것을 권장합니다.
2. **상태 관리**: `state` 객체를 통해 앱의 상태를 관리하며, 변경 시 `saveState()`와 `render()`를 호출하여 UI와 `localStorage`를 동기화합니다.
3. **스타일링**: Tailwind CSS 클래스를 사용하여 반응형 레이아웃(주로 모바일 환경 최적화)을 구현합니다.
4. **언어**: UI 및 소스코드 내 주석은 한국어를 기본으로 사용합니다.
5. **데이터 영속성**: 데이터베이스 없이 `localStorage`에 의존하므로, 브라우저 캐시 삭제 시 데이터가 초기화될 수 있음에 유의하십시오. (리더보드 데이터는 클라이언트 측에서 정적으로 초기화되거나 로컬에만 보존됨)
